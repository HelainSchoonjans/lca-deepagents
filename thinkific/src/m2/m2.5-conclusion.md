## Recap

- Every model call has three parts: **system message**, **tool schemas**, and **messages**
- The system message is assembled once and fully cached — it never changes between turns
- Tool schemas are sent every call but do not change once the agent is initialized
- The **effective message list** changes every turn: new messages are appended and compaction can replace old ones with a summary
- Only what is in these three parts is visible to the model — nothing else
