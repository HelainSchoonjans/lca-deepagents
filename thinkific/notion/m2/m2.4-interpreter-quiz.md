# Quiz: Interpreters

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m2/m2.4-interpreter.html

**Q1: What is the key advantage of Programmatic Tool Calling over normal tool use?**

A) The agent can call more tools per turn
B) Intermediate tool results never enter the LLM context window
C) Tools run faster inside the interpreter
D) The LLM can see all tool results at once

*Answer: B. With PTC, JavaScript drives the tool calls and processes the results entirely inside the interpreter. Only the final computed value is returned to the LLM; the intermediate results never become LLM messages.*

---

**Q2: What language does the agent write when calling the `eval` tool?**

A) Python
B) TypeScript
C) Bash
D) JavaScript

*Answer: D. The `eval` tool runs code in a QuickJS JavaScript runtime. The model writes JavaScript, not Python, as the code argument to `eval()`.*

---

**Q3: Which of the following is available inside the QuickJS runtime?**

A) `fetch` for network requests
B) `fs` for filesystem access
C) `Array.reduce`, `Map`, `JSON`, and `Promise`
D) npm packages via `import`

*Answer: C. QuickJS provides the JavaScript standard library: array methods, `Map`, `Set`, `JSON`, `Math`, `Promise`, and `console.log`. There is no filesystem, network, clock, or package manager access.*

---

**Q4: The interpreter state persists within a thread. What does this mean?**

A) Variables from one `eval()` call are available in the next `eval()` call
B) The interpreter remembers results across different agent invocations
C) JavaScript variables are saved to disk automatically
D) The agent can share interpreter state with other agents

*Answer: A. The QuickJS context lives for the duration of a thread. Variables defined in one `eval()` call remain in scope for subsequent `eval()` calls in the same thread, like a persistent REPL session. Between separate `agent.invoke()` calls, state is also preserved as long as the same `thread_id` is used. Switch to a different `thread_id` and the interpreter starts fresh.*
