# Quiz: Shell and Sandbox Backends

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m2/m2.3-sandboxes.html

**Q1: What does the `execute` tool return to the agent?**

A) stdout, stderr, and an exit code
B) The script file contents
C) A boolean success or failure
D) stdout only

*Answer: A — `execute` returns stdout, stderr, and an exit code as a tool result. The LLM sees all three on the next call, which lets it diagnose errors and retry.*

---

**Q2: What does `root_dir` limit in `LocalShellBackend`?**

A) All shell commands executed via `execute`
B) Filesystem tool access — `ls`, `read_file`, `write_file`, and similar
C) Network access from the agent
D) Which Python packages the agent can import

*Answer: B — `root_dir` scopes the filesystem tools only. The `execute` tool still runs with full host permissions regardless of `root_dir` — that's why `LocalShellBackend` is unsuitable for production.*

---

**Q3: Which sandbox model does this course use?**

A) Agent in Sandbox — the agent runs inside the sandbox
B) Both models equally
C) Neither — the course uses `LocalShellBackend`
D) Sandbox as a Tool — the agent runs outside and calls the sandbox via `execute`

*Answer: D — This course uses the Sandbox as a Tool pattern: the agent runs on the host and calls the sandbox via `execute` and filesystem tools. API keys stay on the host and the agent code is easy to update.*

---

**Q4: How do you make a local file available inside a LangSmith sandbox before the agent runs?**

A) Pass the file path in the system prompt
B) The sandbox automatically syncs the local filesystem
C) Call `backend.upload_files()` with the file contents before invoking the agent
D) Write the file to `/tmp` — the sandbox reads it from there

*Answer: C — `upload_files()` seeds the sandbox filesystem before the agent starts. Without it the sandbox starts empty — the agent has no access to local files unless you explicitly upload them.*
