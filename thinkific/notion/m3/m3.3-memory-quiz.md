# Quiz: Memory

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m3/m3.3-memory-quiz.html

**Q1: How does memory differ from skills in terms of when its content is available to the agent?**

A) Memory is loaded on demand when a task matches; skills are always present
B) Both memory and skills are always present in the system prompt
C) Both memory and skills are loaded on demand when needed
D) Memory is always injected into the system prompt on every call; skills are loaded on demand

*Answer: D*

---

**Q2: What is the difference between agent memory and user memory in Deep Agents?**

A) Agent memory requires a checkpointer; user memory does not
B) Agent memory is read-only; user memory can be written by the agent
C) Agent memory is shared across all users; user memory gives each user their own separate file
D) Agent memory persists across sessions; user memory is cleared after each session

*Answer: C — The distinction is scope.*

---

**Q3: When `memory=["memories/AGENTS.md"]` is passed to `create_deep_agent`, what is created behind the scenes?**

A) A `MemoryMiddleware` that reads the file and injects its content into the system prompt on every call
B) A `FilesystemBackend` rooted at `memories/`
C) A checkpointer that snapshots the file's content between runs
D) A SQLite database for storing and indexing the memory contents

*Answer: A*

---

**Q4: Where in a LangSmith trace can you confirm that memory is being injected correctly?**

A) As a separate Memory tab in the run details panel
B) In the `[agent_memory]` block at the bottom of the system message on every LLM call
C) As the first ToolMessage in every run
D) Only in runs where the agent explicitly reads the memory file

*Answer: B*

---

**Q5: How does an agent persist new information to a memory file in Deep Agents?**

A) By returning a special `memory_update` key in its final response
B) By calling a dedicated `save_memory()` function from the SDK
C) Memory files are read-only; new information requires creating an additional file
D) By calling `edit_file` on one of the paths passed to `memory=[...]`

*Answer: D — This write is visible in LangSmith as an `edit_file` tool call.*
