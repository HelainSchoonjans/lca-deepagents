## MCP: Connecting Agents to External Services

Model Context Protocol (MCP) is an open protocol that standardizes how applications provide tools and context to LLMs. Instead of each integration requiring custom code in your agent, MCP servers expose a consistent interface that any MCP-aware client can connect to.

> 📋 [Embedded slides: MCP architecture walkthrough — see live lesson]

LangChain agents connect to MCP servers using the `langchain-mcp-adapters` library. The adapters wrap each MCP tool as a standard LangChain `BaseTool`, so the agent sees it identically to any locally-defined tool.

### Configuration

deepagents discovers MCP servers from `.mcp.json` in your project root (or `~/.deepagents/.mcp.json` for user-level config). The project repo already includes a config pointing at the LangChain documentation server:

> 📋 [Embedded slides: .mcp.json configuration diagram — see live lesson]

Load those tools in Python with `get_mcp_tools`:

```python
# python/m1/m1.6_scratch_agent_mcp.py
from deepagents_code.mcp_tools import get_mcp_tools

tools, session_manager, server_infos = await get_mcp_tools(".mcp.json")
```

Pass `tools` to `create_deep_agent` as you would any other tool list. When you are done, call `await session_manager.cleanup()` to close the connections.

**Transport details**

There are three transport types:

- **stdio** runs the server as a local subprocess. Messages pass as JSON through stdin/stdout. No networking required.
- **HTTP** sends a standard HTTPS request to a remote server and waits for the response. A new connection opens for each tool call.
- **SSE** (Server-Sent Events) opens a persistent HTTP connection. The server can push events back in real time, useful for long-running tools or streaming results.

> 📋 [Embedded slides: transport types walkthrough — see live lesson]

In `.mcp.json`, set `"type"` to `"http"` or `"sse"` for remote servers, or use `"command"` and `"args"` for stdio servers.

### Authorization

Some MCP servers require the user to authenticate before the agent can call their tools.

> 📋 [Embedded slides: authorization walkthrough — see live lesson]

Add `"auth": "oauth"` to the server entry in `.mcp.json`:

```json
{
  "mcpServers": {
    "gmail": {
      "type": "http",
      "url": "https://gmailmcp.googleapis.com/mcp/v1",
      "auth": "oauth",
      "allowedTools": ["search_threads", "get_thread"]
    }
  }
}
```

When deepagents detects a server that requires OAuth, it pauses and opens a browser window for the user to log in. Tokens are stored in `~/.deepagents/.state/mcp-tokens/` and reused on subsequent runs.

`allowedTools` lets you restrict which tools from a server the agent can see.

### Human-in-the-Loop (HITL)

Even after authorization, you may want a human to confirm before the agent reads or writes data. Pass `interrupt_on` to `create_deep_agent`:

> 📋 [Embedded slides: HITL walkthrough — see live lesson]

```python
agent = create_deep_agent(
    model=model,
    tools=tools,
    interrupt_on={
        "search_threads": True,
        "get_thread":     True,
    },
    checkpointer=MemorySaver(),
)
```

`True` pauses the agent before calling that tool and presents the pending call to the user.

> **`interrupt_on` requires a checkpointer.** When the agent pauses for human approval, it needs to save its current state so execution can resume from exactly the same point after a decision is made. `MemorySaver` keeps state in memory and is fine for development; use a persistent checkpointer in production.

## Connecting to a Docs Server

This lab connects an agent to the LangChain documentation server, an HTTP MCP server maintained by LangChain, not you.

### Step 1: Load and inspect

Open `python/m1/m1.6_scratch_agent_mcp.py`. The first thing `main()` does is load the MCP tools and print each tool's name and description:

```python
# python/m1/m1.6_scratch_agent_mcp.py
import sys
import asyncio
import json
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))

from models import model
from deepagents import create_deep_agent
from deepagents_code.mcp_tools import get_mcp_tools

MCP_CONFIG = Path(__file__).resolve().parent / ".mcp.json"

MCP_CONFIG.write_text(json.dumps({
    "mcpServers": {
        "docs-langchain": {
            "type": "http",
            "url": "https://docs.langchain.com/mcp"
        }
    }
}, indent=2))


async def main():
    tools, session_manager, server_infos = await get_mcp_tools(str(MCP_CONFIG))

    for info in server_infos:
        print(f"\n{info.name} ({info.transport}): {len(info.tools)} tool(s)")
        for t in info.tools:
            print(f"  {t.name}")
            print(f"  {t.description[:90]}")


asyncio.run(main())
```

```

```

### Step 2: Ask a question

After printing the tools, the agent is invoked:

```python
agent = create_deep_agent(model=model, tools=tools)

result = await agent.ainvoke({
    "messages": [{"role": "user", "content": "How does LangGraph's interrupt API work, and what role does a checkpointer play?"}]
})
print(result["messages"][-1].content)
```

Open LangSmith. The agent calls `docs-langchain_search_docs_by_lang_chain` one or more times, reads the results, and synthesizes an answer.

### Step 3: Restrict with `allowedTools`

`allowedTools` is already set in `m1.6_scratch_agent_mcp.py`, restricting the server to the search tool only:

```python
MCP_CONFIG.write_text(json.dumps({
    "mcpServers": {
        "docs-langchain": {
            "type": "http",
            "url": "https://docs.langchain.com/mcp",
            "allowedTools": ["search_docs_by_lang_chain"]
        }
    }
}, indent=2))
```

> **`allowedTools` vs. `disabledTools`:** `allowedTools` is an allowlist; only the named tools are kept. `disabledTools` is a denylist; only the named tools are removed. Use `allowedTools` when you want a strict, stable subset.

## Recap

- MCP is an open protocol that standardizes how applications expose tools and context to LLMs
- `langchain-mcp-adapters` wraps each server tool as a standard `BaseTool`; the agent cannot tell MCP tools apart from locally-defined ones
- Servers are configured in `.mcp.json`; the `type` field sets the transport (`stdio`, `http`, or `sse`)
- `allowedTools` restricts which tools from a server the agent can see
- `auth: "oauth"` triggers a one-time browser authorization; tokens are cached and reused automatically
- `interrupt_on` pauses the agent before a named tool call for human approval; requires a checkpointer
