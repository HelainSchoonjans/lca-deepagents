# Quiz: MCP

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m1/m1.6-mcp-quiz.html

**Q1: How does `langchain-mcp-adapters` expose MCP server tools to the agent?**

A) As a special MCP tool type the agent calls differently from normal tools
B) As raw JSON schemas the agent parses on each call
C) As a separate tool registry the agent queries at startup
D) As standard LangChain `BaseTool` objects; the agent cannot tell them apart from locally-defined tools

*Answer: D*

---

**Q2: Which MCP transport type keeps a persistent connection open and supports streaming results?**

A) SSE (Server-Sent Events): opens a persistent connection the server can push events over
B) stdio: runs the server as a local subprocess
C) HTTP: sends a new request for each tool call
D) All three transports support persistent connections

*Answer: A*

---

**Q3: What does the `allowedTools` field in `.mcp.json` do?**

A) Grants the agent permission to execute those tools without human approval
B) Specifies the tool schemas to use instead of fetching them from the server
C) Restricts which tools from that server the agent can see
D) Sets a rate limit on how often each tool can be called

*Answer: C*

---

**Q4: When deepagents encounters `auth: 'oauth'` in `.mcp.json` for the first time, what happens?**

A) The agent attempts to call the server anonymously and retries with credentials on failure
B) An error is thrown until the user manually adds a token to `.mcp.json`
C) The server is skipped and its tools are marked unavailable
D) deepagents pauses and opens a browser window for the user to authorize; tokens are saved and reused on subsequent runs

*Answer: D*

---

**Q5: You add `interrupt_on` to `create_deep_agent` for an MCP tool but forget to pass a checkpointer. What happens?**

A) The agent runs normally and ignores the `interrupt_on` setting
B) An error is raised at invocation time because `interrupt_on` requires a checkpointer to save and restore state
C) The agent pauses but cannot resume; graph state is lost
D) The interrupt fires once, then is automatically disabled for that session

*Answer: B*
